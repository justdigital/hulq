HulQ A simple yet powerful XSLT framework

HulQ smashes your XSLT file into many small files, so you have one XSLT per template tag.

After changes to the files, HulQ rebuilds the template again with minified assets.

Get started easily

  1. git clone https://google.com/hulq.git your_project #Clone the repo and choose a name
  2. cd your_project #Go into your created folder
  3. npm install #Installs the NPM dependencies
  4. sudo chmod +x ./hulq #Give exec permission on the hulq file
  5. mv /path/to/my_xslt_frontend_file.xslt resources/ #Move your current template file to the resources folder so it can be smashed
  6. ./hulq smash my_xslt_frontend_file #Break your template in many parts (you don't have to specify the file extension)
  7. ./hulq prepare #Move your XSLT template to the project folder (overwrites previously preparated files)
  8. #Now it is your turn: You can edit the xslt files (workspace/**/*.xslt)
  9. ./hulq recover #Create the brand new xslt file with everything together
  10. ./hulq deploy* #*Comming soon: Deploy your recovered xslt to the GSA control panel
