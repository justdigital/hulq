A simple yet powerful XSLT framework

HulQ smashes your XSLT file into many small files, so you have one XSLT per template tag. After changes to the files, HulQ rebuilds the template again with minified assets.

HulQ Smash!

To feel the power of HulQ you just need to follow 6 easy steps. Enjoy it, and get your work more simple and productive.

Hey... more important... feel free to contribute to become HulQ more green and strong.

We love more engineering helping us to do more for opensource technologies.

1. Install HulQ npm package. It needs to be globally available. 
$ sudo npm install hulq -g
2. Navigate to your project's directory. 
$ cd your_project_directory
3. Create a new HulQ project based on your_template.xslt file (extension can be ommited). 
$ hulq begin your_template.xslt
4. Now it is your turn: You can edit the xslt files (workspace/**/*.xslt).
5. Create the brand new xslt file with everything together. 
$ hulq recover
6. Create the brand new xslt file with everything together. 
$ hulq deploy
